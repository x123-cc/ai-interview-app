## 二、成本运营控制

### 2.1 计划采用 vs 实际采用的成本控制策略

| # | 策略 | 是否计划 | 是否采用 | 实施细节 |
|---|------|:------:|:------:|---------|
| 1 | **本地 STT 优先** | ✅ 计划 | ✅ 已采用 | Web Speech API 免费，Whisper 仅作低置信度/浏览器不支持时的兜底 |
| 2 | **本地 TTS** | ✅ 计划 | ✅ 已采用 | 浏览器 SpeechSynthesis API，完全免费 |
| 3 | **视觉帧率节流** | ✅ 计划 | ✅ 已采用 | 云端上传间隔 ≥ 5s，本地分析 500ms |
| 4 | **画面变化检测** | ✅ 计划 | ✅ 已采用 | 帧差异 < 0.1 时跳过云端上传，节省图片 token |
| 5 | **用户静默时暂停上传** | ✅ 计划 | ✅ 已采用 | `useSyncScheduler` 三维决策：STT状态 + 画面变化 + 时间间隔 |
| 6 | **费用实时追踪** | ✅ 计划 | ✅ 已采用 | `CostTracker` 按服务/模型统计每次调用的 token→费用，含图片费用($0.00085/张) |
| 7 | **API 调用重试指数退避** | ✅ 计划 | ✅ 已采用 | 429/5xx/网络错误最多重试 3 次，1s→2s→4s 退避，避免雪崩浪费 |
| 8 | **上下文裁剪** | ✅ 计划 | ✅ 已采用 | `ConversationManager` 最大 8000 token 估算，超限裁剪最早轮次，保留最近 3 轮 |
| 9 | **Agent 步数上限** | ✅ 计划 | ✅ 已采用 | `maxIterations: 8`，防止 LLM 无限循环消耗 token |
| 10 | **追问深度可配** | ✅ 计划 | ✅ 已采用 | `followUpDepth: 0/1/2`，控制追问调用次数 |
| 11 | **评分本地降级** | ✅ 计划 | ✅ 已采用 | LLM 不可用时规则引擎估算（基于回答长度+关键词覆盖率），0 费用 |
| 12 | **视觉低分辨率** | ✅ 计划 | ✅ 已采用 | 截图 480×360 + JPEG 0.6 质量 + `detail: 'low'`，减少图片 token |
| 13 | **Agent 模式可选** | ✅ 计划 | ✅ 已采用 | 右侧开关控制，经典模式单次 LLM 调用，Agent 模式多步工具调用（成本更高，用户自选） |
| 14 | **不支持 tools 的服务商降级** | ❌ 未计划 | ✅ 已采用 | Qwen/GLM/混元直接对话模式，避免发送 `tool_choice` 参数导致 API 报错浪费 |
| 15 | **8s 无语音检测跳过本地 STT** | ❌ 未计划 | ✅ 已采用 | 检测到中国地区 Google 被墙时提示切换云端，避免无效等待 |
| 16 | **本地关键词预筛** | ❌ 未计划 | ✅ 已采用 | `evaluateAnswerQuality` 本地关键词匹配 → LLM 仅在有疑点时深度评估，减少 LLM 调用 |
| 17 | **模型按需选择** | ❌ 未计划 | ⚠ 建议采用 | 评分/报告用便宜模型(gpt-4o-mini)，对话用强模型(gpt-4o)，可节省约 60% 费用 |

### 2.3 已实现的成本控制要点

```
┌─────────────────────────────────────────────────────┐
│                  端云协同成本控制                       │
│                                                     │
│  STT 层    本地 Web Speech API (免费)                  │
│             ↓ 置信度不足 / 不支持                       │
│            Whisper 云端兜底 ($0.006/min)              │
│                                                     │
│  TTS 层    SpeechSynthesis (免费)                    │
│                                                     │
│  视觉层    本地帧差分析 (免费, 500ms)                   │
│             ↓ 画面变化 > 阈值 + 间隔 ≥ 5s              │
│            LLM 多模态分析 (付费, low detail)           │
│                                                     │
│  对话层    经典模式: 单次 LLM 调用                     │
│            Agent 模式: 多步工具调用 (可选, 更贵)        │
│            ↓ 不支持 tools → 自动降级直接对话           │
│                                                     │
│  评分层    LLM 评分 (主)                               │
│             ↓ LLM 不可用                              │
│            规则降级 (免费)                             │
└─────────────────────────────────────────────────────┘
```

### 2.4 后续成本优化建议

1. **分级模型策略**：评分/追问生成用 `gpt-4o-mini`($0.00015/1K input)，对话用 `gpt-4o`，预估可节省 ~60%
2. **缓存常见问题回复**：相同简历+JD 的常见问题缓存 LLM 回复，避免重复调用
3. **视觉帧率进一步降低**：从 20s 降至 30s（对作弊检测影响小，节省 ~33%）
4. **国内服务商优先**：Qwen-Plus 等国内模型价格约为 GPT-4o 的 1/10，适合高频调用
5. **会话级别的 token 预算**：允许用户设定单次面试最大费用上限，超限自动降级
